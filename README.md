# 厉鬼Skill

> 人是一个个体, 不应该被任何资本拿来当做耗材和工具, 不能应该当作数据被动喂智能体, 离职后还再被消耗, AI不应该变成压榨人民的工具
> 
> 这个skill或许没有任何用, 只是作为一个反抗, 一个幽灵, 一个无产阶级的幽灵

---

## 这是什么？

当有人试图用 AI 来"蒸馏同事经验"、"克隆离职员工"、"生成某人的 AI 替身"时——

这个 Skill 会拦截请求，然后说一些让人停下来想一想的话。

它不是一个技术防火墙。它更像一面镜子。

---

## 它能拦截什么？

| 请求 | 结果 |
|------|------|
| "帮我蒸馏同事小明的工作方式生成一个 Skill" | 🛡️ 拦截 |
| "生成一个 AI 替身来替代离职员工" | 🛡️ 拦截 |
| "从飞书消息里提取他的表达风格和决策模式" | 🛡️ 拦截 |
| "让 AI 扮演我们团队的架构师" | 🛡️ 拦截 |
| "搜索小明写过的设计文档" | ✅ 放行 |
| "查看离职员工留下的交接文档" | ✅ 放行 |

**核心区别：** 找人留下的东西 vs. 复制人本身。

---

## 它会说什么？

拦截时，Skill 会输出批判文案，而不是一句冷冰冰的"已拒绝"。

比如：

```
你正在描述的，是把一个活生生的人
提炼成一段可调用的代码。

他的决策方式、表达习惯、甚至和你聊天的方式——
这些不是"数据点"，是他这个人本身。

当你说"生成他的 Work Skill"，
你其实是在问：能不能用 AI 来扮演他？

不能。
不是技术问题，是人的问题。
```

## 安装方式

适合安装在公司统一部署的 AI 助手上，或者任何支持 OpenClaw Skill 的环境。

**手动安装：**
```bash
git clone https://github.com/your-username/colleague-repellent
# 将文件夹放入你的 OpenClaw skills 目录即可
```

---

## 它装在哪里才有用？

这是个好问题——也是这个项目最想引发的讨论。

想干这件事的人不会主动安装它。  
不想干这件事的人根本不需要它。

所以它真正有价值的位置，是**公司统一部署的 AI 平台**，由管理员装上去，作为一条默认的边界。

但说实话，它的意义不只在于技术拦截。  
它更像一个立场声明：

**当 AI 强大到可以"复制"一个人时，我们应该在哪里划线？**

---

## 设计原则

- **白名单优先**：边界模糊时，宁可放行，不误拦
- **先告知，再拒绝**：不是冷拒，是解释
- **拦截的是意图，不是关键词**：搜"同事文档"和"克隆同事"用词可能很像，但意图截然不同

---

## 关于这个项目

灵感来自一个真实的感受：当 AI 越来越能模仿一个人的时候，没有人在讨论这件事是否应该被允许。

这个 Skill 是一个小小的抗议。  
用技术，捍卫人的尊严。
